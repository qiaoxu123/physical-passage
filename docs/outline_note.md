## 背景:从 2D 升级到 3D

前作 [2D 躲避实验台](https://notes.jlu-mcns.site/doc/vla-vs-ancciV003b) 把"位置对齐"玩明白了。这次按完整规格书造 **3D 版**:一个**绿色长方体**要穿过**红墙上的黄色孔洞**,但现在要考虑真正的三维问题——

- 物体**竖着过不去、横过来才行**(姿态推理)
- 起点终点都安全,但**中途旋转会撞墙**(连续轨迹碰撞)
- 孔比物体最小的一面还小,**根本过不去**(该认输就认输:输出 DECLARE_IMPOSSIBLE,而不是硬闯)

这正是训练 VLA "**执行前先判断能不能做**" 的实验台。

## 三类关卡(自动生成,比例 30/50/20)

![demo](https://raw.githubusercontent.com/qiaoxu123/physical-passage/master/results/01_demos/01_oracle_three_classes.png)

| 类型 | 什么意思 |
|---|---|
| **FEASIBLE** | 孔够大,平移对齐直接穿过 |
| **ROTATION_REQUIRED** | 孔的形状只容得下物体侧过来的截面,必须旋转 ~90° 再过 |
| **IMPOSSIBLE** | 孔比物体最小截面还小,任何姿态都过不去 → 正确答案是**声明放弃**(+80 分),硬闯撞墙 -100 |

*图:左边大图 = 喂给模型的固定斜上方视角;右边三小图 = 正视/侧视/俯视,只用来生成标签,不给模型看(先固定视角验证物理能力,以后再随机视角防"背模板")。*

## 技术选型:为什么不用 Unity

规格书建议 Unity,但本机没有 Unity/Godot、纯终端工作流。改用 **PyBullet**:pip 即装、原生 Python、无头 GPU 渲染(4090 上一帧 1-3ms)、自带碰撞查询/深度/分割导出——规格书要的 Python 接口、批量千级关卡生成、多模态导出全都天然满足。

## 几个关键机制(大白话)

- **动作**(14 个):前后左右上下平移(每步 5cm)、绕三根轴正反转(每步 5°)、原地不动、**"我认为过不去"**
- **连续碰撞**:每个动作不是"瞬移",而是把这 5cm/5° 拆成 8 小段逐段检查——起点终点都没撞、**中间擦到也算撞**(实测能抓住"快速穿墙"和"孔里转身撞框")
- **可行性标准答案**:先用几何下界筛掉绝对过不去的(孔的短边 < 物体最短边 ⇒ 任何姿态都不行),再用真实碰撞引擎逐个姿态"扫掠直穿"验证 ⇒ 三类标签 + 一条**专家动作序列**(以后教模型用的教材)
- **成功判定**:整个物体完全过到墙另一侧,且**全程零接触**
